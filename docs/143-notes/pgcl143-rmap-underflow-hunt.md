Then `systemd[1]` segfaults **11× at the SAME addr 0x744000000dc / ip 0x7065c884f165** → kill init. **CORRECTION (same day): those 184 WARNs are FALSE POSITIVES, not the bug.** Checked both sides: `try_to_migrate_one` order-0 LOOPS `nr_pages` `folio_remove_rmap_pte` (rmap.c:369, comment "order-0: per-PTE mapcount; loop the rmap drop") AND `remove_migration_pte` order-0 LOOPS `nr_pages` `folio_add_anon_rmap_pte` (migrate.c:514).
